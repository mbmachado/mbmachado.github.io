Padrões de Codificação Java
Motivação
  • Validação
  • Comunicação
  • Leitura
  • Mautenção
Comentários
As classes e interfaces devem ser precedidas de comentários para indicar sua função e seu autor

/**
 * Lida com a comunicação com banco de dados
 * @author autor
 */
public class Connection {

}
Comentários curtos podem ser feitos na mesma linha do código e comentários curtos seguidos devem ser alinhados

if(a == 2) {
	max = 4; /* Comment  */
	return TRUE; /* Comment  */

	max = 4;        /* Comment  */
	return TRUE;	/* Comment  */
}
O comentário // não deve ser utilizado duas vezes seguidas, exceto para comentar linhas de código

//System.out.println("email = "+param2[0]+" senha = "+param2[1]);
//System.out.println("nome = "+param2[0]+" cpf = "+param2[1]);
Pacotes e Importes
Primeiro devem ser declarados os pacotes e posteriormente os importes. Evitar importações com *, para não obter classes que não serão utilizadas

package componente;

import java.awt.*;                     /*Errado*/
import javax.swing.*;

import java.awt.event.ActionEvent;     /*Certo*/
import javax.swing.JButton;
Composição das classes e interfaces
Variáveis, método construtor e demais métodos

public static double var1;
protected static String var2;
private static int var3;

public double var4;
protected String var5;
private int var6;
Apresentação
Identação deve ser feita com 4 espaços ou 1 Tab

if(a == b) {
	if( a == c) {
		System.out.println("OK");
	}
}
Os parênteses de abertura da implementação de classes, métodos, blocos condicionais e blocos de loop devem estar na mesma linha

if(a == b)      /*Errado*/
{
	
}

if(a == b) {    /*Correto*/

	
}
Evitar linhas de código com muitos caracteres (não mais que 70)
Quebrar em pontos específicos

longName1 = longName2 * (longName3 + longName4 - longName5)
           + 4 * longname6;
 
longName1 = longName2 * (longName3 + longName4
                       - longName5) + 4 * longname6;
Separar termos de operações aritméticas e de comparação com 1 espaço em branco

var2= var1+4 ;     /*Errado*/

var2 = var1 + 4;   /*Correto*/
Declarações
Efetuar a declaração de variáveis do mesmo tipo em uma mesma linha e inicializar depois da declaração

int var1;             /*Errado*/
int var2;
int var3;     

int var1, va2, va3;   /*Correto*/
Nomeação
Variáveis devem ter nome curto e intuitivo, para facilitar a identificação da sua função. Evitar caracteres ‘_’ e ‘$’ e seguir a prática CamelCase

int umVar, outraVar, maisUmaVar;
Métodos devem seguir a prática CamelCase e a primeira palavra deve ser um verbo no imperativo

public Component getTableCellRendererComponent() {

}
Classes e Interfaces devem seguir a prática CamelCase e as palavras devem ser substantivos no singular

public class ButtonColumn extends AbstractCellEditor {
	
}
Constantes devem ser em maiúsculo e com palavras separadas por underscore “_”

public final int MAX_VALUE;
public final int MIN_VALUE;
Pacotes devem seguir o modelo br.com.nome_da_empresa.nome_do_pacote

package com.sun.event;
package com.google.charts;
